# Структура проекта

Весь код (помимо модели нейросети) написан на языке C с использованием фреймворка espidf. 
Конфигурация микроконтроллера подобрана для ESP32-S3. 
Все это сделано, с целью иметь лучший контроль над аппаратной частью.
К тому же espidf родной фреймворк ESP32

1. **/src**
  Здесь "main.c" — точка входа в программу. Этот скрипт объединяет в себе заголовочные файлы из **/include** и **/lib**.
  Он написан на операционной системе FREERTOS — работа системы реализована через задачи, прерыватели и семафоры.
  
2. **/include**
  Здесь есть заголовочный файл "app_config.h". Он подключает все необходимые библиотеки для всех остальных заголовочных файлов и "main.c".
  Также в нем содержатся определения констант через макросы и объявления глобальных переменных для исходного кода и заголовочных файлов.

3. **/lib**
  Здесь есть все заголовочные файлы и исходные коды отдельных частей программы, объединенные в библиотеки:
  модель нейросети, конфигурация сторожевого пса (watchdog) и реализации функций для работы с интерфейсами

*В других README.md есть другая важная информация для разработчиков.*
*Помимо этого непустой README.md находится в директории **/lib/audio**, где есть небольшая документация для i2s интерфейса.*

# Этап 1: (Выполнен)

**Цель: собрать рабочий диктофон для сбора датасета на полигоне с дронами и получить его.**

**Что было сделано:**

1.  Получили базовый опыт по работе с esp32 и интерфейсами
2.  Написали простой скрипт для записи аудио
3.  Получили первый жизнеспособный диктофон, решив проблемы с питанием устройства
4.  Довели питание до ума, приведя аппаратную конфигурацию в порядок
5.  Довели программу до ума, перенеся его в FREERTOS для большего аппаратного контроля, надежного отклика и относительно удобного разделения логики
6.  Все распаяли и дали устройству корпус, ветрозащиту и питание от аккумулятора, обеспечив надежность электроники

**Как работает диктофон?**

1.  Он подключается к питанию от аккумулятора БПЛА. Может быть слышен треск от индукции, но это естественный звук для такого питания.
2.  Нажимается кнопка ON/OFF и идет инициализация "подсистем". По завершении мигает красный светодиод в случае исправности.
3.  Далее можно нажать на кнопку "Record". Зеленый светодиод при успехе загорится и пойдет запись звука.
4.  Запись звука идет по порядку, от номера последнего сохраненного аудиофайла в директории **sdcard/audio**.
    Там же хранятся все записанные ранее аудиофайлы и текстовый файл, хранящий номер последнего записанного аудиофайла, 
    откуда достается номер последнего записанного файла. 
    Если директории или текстового файла нет, они создаются, а номер будет 000000 (Начинается с 000001).
    (Так сделано, чтобы не перебирать номера в названии аудиофайлов)
5. Запись останавливается повторным нажатием на кнопку "Record".
6. Система корректно выключается только при повторном нажатии на кнопку "ON/OFF" и отсутствии процесса записи.
7. Можно повторять по циклу. Можно нажимать на любые кнопки и не бояться, что система сломается. Но делать этого не рекомендуется на всякий случай.

**Зачем знать диктофон?**

Следующее первичное устройство будет иметь основу от нашего конечного диктофона.
  
# Этап 2: (В процессе)

**Цель: получить первый рабочий вариант детектора БПЛА**

**Рекомендации: ознакомиться с FREERTOS на более тривиальных примерах за счет запасного ESP32-S3 в комплекте "железа"**
              **ознакомиться с алгоритмом задержек и изучить i2s интерфейс поподробнее** 

**Задачи**

1.  Получить недостающий микрофон ICS43434 и раму для квадрокоптера (Длина стороны, вероятно, 40 см). 
    На раме в четырех вершинах будут микрофоны, а посередине все остальное железо.
    Можно попросить куратора перенести один микрофон на диктофоне в одну из вершин и установить 3 другие с проводкой
2.  Дополнить библиотеку интерфейса i2s для 4 микрофонов (2 пары в стерео -> +1 программный канал i2s). 
    Плюс, вероятно, надо будет увеличить частоту дискретизации до 44,1 КГц для значительной разницы в задержках микрофонов.
3.  Там же реализовать функции для алгоритма выявления направления звука по задержкам в микрофонах.
    Желательно провести разделение логики. То есть, для этого алгоритма в директории **/lib/audio** создать новый заголовочный файл и исходный код к нему.
4.  Самое неприятное. Переписать точку входа программы для тестирования алгоритма.
    Скорее всего, нужно будет удалить кнопку "Record" из программы и удалить логику для зеленого светодиода.
    Он нужен для индикации процесса записи. Если произошла ошибка при нажатии, он не засвечивается.
    
    Теперь задача записи будет автономной, и добавится задача алгоритма задержек, чтобы было разделение логики.
    ! На самом деле, тогда придется подраться с семафорами. Должен добавиться один семафор.
    
    Отлично, у нас есть обновленная система, и мы можем определить направление звука.
    ! Вопрос. Скорее всего, сторонние шумы будут перебивать определение направления, поэтому алгоритм нужно сделать умнее или полностью его сменить.
    ! Это возможно, если дрон летит далеко, в радиусе ~200-300 метров.
    ! Возможно, при анализе от нейросети можно будет получить одно конкретное направление. Но пока без этого.

    Но нужно как-то временно сохранить wav с записанным интервалом для дальнейшей обработки нейросети,
    предварительно превратив данные с четырех микрофонов в сигнал на частоте дискретизации 16 КГц.
    ! Возможно, можно кормить нейросеть непрерывным потоком из pcm буфера, запустив нейросеть на втором ядре, но это вряд ли,
    ! так как ему, наверное, нужна полноценная спектрограмма
    ! но пока достаточно записать файл в карту через библиотеку wav_writer.
    ! В дальнейшем можно попробовать хранить аудио для анализа в PSRAM.

    Но если модели нейросети нет? (К сожалению, с нашим единственным специалистом может случиться задержка) 
    Можно создать отдельную задачу для нейросети и добавить туда "затычку" в виде задержки vTaskDelay(pdMS_TO_TICKS(3000)).
    Можете установить другую удобную задержку помимо 3000.

    Получим следующую последовательность.

**Включение -> Инициализация -> (ц)Записать звук какой-то длительности -> Определить направление ->**
**Сохранить аудио -> Запустить нейросеть -> Получить результат-> (ц) или деинициализация -> Выключение**

В действительности, деинициализация может произойти в любой момент времени из-за прерывателя ISR.
